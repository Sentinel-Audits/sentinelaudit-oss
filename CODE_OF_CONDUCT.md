# Code of Conduct

## Our Standard

We want the SentinelAudit open-source surface to be useful, technically honest,
and respectful to contributors of all experience levels.

Examples of behavior that help build that environment include:

- being respectful and professional
- giving technical feedback without hostility
- assuming good intent while still being precise
- welcoming questions, corrections, and thoughtful disagreement

Examples of unacceptable behavior include:

- harassment, intimidation, or personal attacks
- insulting or dismissive language
- bad-faith misrepresentation of another person's work or intent
- publishing private, sensitive, or security-relevant material irresponsibly

## Scope

This Code of Conduct applies within the public SentinelAudit repository,
issues, pull requests, and project discussions.

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, commits, or
other contributions that violate this Code of Conduct.

## Reporting

If you experience or observe unacceptable behavior, report it privately to the
project maintainers rather than escalating publicly.
